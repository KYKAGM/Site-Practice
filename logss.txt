Документация по разработке проекта "Сөзді тап! (Казахский Wordle)"

Шаг 1: Изучение архитектуры проекта и файла README.md.
Шаг 2: Установка зависимостей бэкенда (Django, djangorestframework, psycopg2 и др.) в виртуальное окружение.
Шаг 3: Настройка базы данных. В файл backend/.env добавлен пароль от PostgreSQL (helloworld).
Шаг 4: Запуск локального сервера PostgreSQL и создание базы данных kazakh_wordle с помощью скрипта.
Шаг 5: Выполнение миграций базы данных (python manage.py makemigrations game, python manage.py migrate).
Шаг 6: Загрузка словарной базы (119 слов из JSON файлов) в базу данных (python manage.py load_words).
Шаг 7: Инициализация фронтенд-приложения на базе React и Vite (npx create-vite).
Шаг 8: Установка npm-зависимостей фронтенда, включая Tailwind CSS v3 и иконки lucide-react.
Шаг 9: Настройка конфигурации Tailwind (tailwind.config.js) и создание глобальных стилей (index.css) с кастомными анимациями (flip, pop, shake).
Шаг 10: Настройка прокси-сервера в vite.config.js для перенаправления API-запросов на порт 8000 (Django бэкенд).
Шаг 11: Разработка логики игры (API клиент gameApi.js, кастомный хук useGame.js).
Шаг 12: Создание пользовательского интерфейса (компоненты Header, LengthSelector, LetterTile, WordRow, GameBoard, Keyboard, HintButton, GameModal, HelpModal и сборка в App.jsx).
Шаг 13: Обновление файла index.html (установка языка kk, добавление meta-описания и заголовка для SEO).
Шаг 14: Успешная сборка проекта (npm run build) для проверки отсутствия ошибок.
Шаг 15: Предоставление обходной команды для ошибки Execution Policy в PowerShell при запуске npm run dev.

Шаг 16: Изучен новый запрос из приложенного текста: улучшить дизайн, добавить настройки, светлую/темную тему, режим для дальтоников, кнопку инструкции, декоративный фон и кастомный курсор.
Шаг 17: Проверена структура frontend-проекта и текущие компоненты React/Vite без установки новых библиотек.
Шаг 18: Добавлены компоненты SettingsPanel, SideRays и TargetCursor для панели настроек, фоновых лучей и кастомного курсора.
Шаг 19: Обновлен App.jsx: подключены настройки темы, сохранение в localStorage, режим для дальтоников, фоновые лучи и курсор.
Шаг 20: Обновлены Header, LengthSelector, HintButton, Keyboard, LetterTile, HelpModal и GameModal: исправлены видимые тексты, добавлены новые CSS-классы и состояния интерфейса.
Шаг 21: Полностью обновлен frontend/src/index.css: добавлены CSS-переменные тем, адаптивная верстка, стили плиток/клавиатуры, модальные окна, фоновые лучи, кастомный курсор и colorblind-режим.

Шаг 22: Проверена сборка фронтенда командой cmd /c npm run build; Vite успешно собрал проект без ошибок.

Шаг 23: Попытка открыть локальный интерфейс через in-app браузер была остановлена: браузер блокировал localhost/127.0.0.1, а фоновые варианты запуска dev-сервера в этом окружении сразу завершались.
Шаг 24: По просьбе пользователя визуальная проверка перенесена на ручной запуск и проверку пользователем позже.

Шаг 25: Изучен новый запрос: добавить кнопку быстрого проигрыша, перевести интерфейс на казахский, отметить jarjan/qate.js, заменить курсор на React Bits TargetCursor, улучшить меню настроек, добавить ripple-кнопки букв и плавный переход светлой/темной темы.
Шаг 26: Проверен приложенный текст с полным кодом TargetCursor React Bits и зависимостью gsap.

Шаг 28: Установлена зависимость gsap во frontend для полноценного курсора React Bits TargetCursor.

Шаг 29: Добавлен backend endpoint /api/game/give-up/ для быстрого проигрыша с раскрытием ответа.
Шаг 30: Обновлены frontend API и useGame: добавлен giveUp, исправлены казахские сообщения ошибок и нормальная проверка казахских букв.
Шаг 31: TargetCursor заменен на React Bits-версию с gsap и отдельным файлом TargetCursor.css.
Шаг 32: Интерфейс Header, SettingsPanel, LengthSelector, HintButton, HelpModal и GameModal переведен на казахский язык.
Шаг 33: Добавлены кнопка "Жеңілу", ripple-эффект для клавиатуры, более интересное меню настроек и плавные переходы темы в index.css.
Шаг 34: Добавлен файл backend/game/data/DICTIONARY_SOURCES.md с отметкой репозитория jarjan/qate.js, лицензии MIT и пояснением, как он связан с текущей проверкой слов.

Шаг 35: Исправлены поврежденные казахские сообщения ошибок в backend/game/views.py и вывод команды load_words.py.

Шаг 36: Исправлены поврежденные казахские строки в backend/game/models.py и backend/game/ai_hints.py.
